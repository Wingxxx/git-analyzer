# Git-Analyzer Skill

## 功能概述

Git-Analyzer是一个强大的Git仓库分析工具，提供以下功能：

1. **仓库管理**
   - 检测本地路径是否为Git仓库
   - 克隆远程Git仓库
   - 更新本地Git仓库
   - 列出仓库分支
   - 切换分支

2. **提交记录分析**
   - 获取提交记录（支持按时间范围过滤）
   - 获取提交详细信息

3. **贡献度计算**
   - 分析开发者贡献度
   - 基于多种指标计算贡献得分
   - 生成贡献度排名

4. **报告生成**
   - 生成Markdown格式的分析报告
   - 包含贡献度排名、最近提交记录和团队整体分析

## 使用方法

### 命令格式

```
git-analyzer <command> [options]
```

### 可用命令

#### 1. status - 检查仓库状态

**功能**：检测指定路径是否为Git仓库

**用法**：
```
git-analyzer status [path]
```

**参数**：
- `path`：仓库路径，默认为当前目录

**示例**：
```
git-analyzer status
```

#### 2. clone - 克隆仓库

**功能**：从远程地址克隆Git仓库到本地

**用法**：
```
git-analyzer clone <url> <path>
```

**参数**：
- `url`：仓库远程地址
- `path`：本地保存路径

**示例**：
```
git-analyzer clone https://github.com/user/repo.git ./my-repo
```

#### 3. update - 更新仓库

**功能**：更新本地Git仓库到最新版本

**用法**：
```
git-analyzer update [path]
```

**参数**：
- `path`：仓库路径，默认为当前目录

**示例**：
```
git-analyzer update ./my-repo
```

#### 4. branches - 列出分支

**功能**：列出仓库的所有分支

**用法**：
```
git-analyzer branches [path]
```

**参数**：
- `path`：仓库路径，默认为当前目录

**示例**：
```
git-analyzer branches
```

#### 5. switch - 切换分支

**功能**：切换到指定分支

**用法**：
```
git-analyzer switch <branch> [path]
```

**参数**：
- `branch`：分支名称
- `path`：仓库路径，默认为当前目录

**示例**：
```
git-analyzer switch develop
```

#### 6. commits - 查看提交记录

**功能**：获取仓库的提交记录，支持按时间范围过滤

**用法**：
```
git-analyzer commits [path] [--since <date>] [--until <date>]
```

**参数**：
- `path`：仓库路径，默认为当前目录
- `--since`：开始日期，格式为YYYY-MM-DD
- `--until`：结束日期，格式为YYYY-MM-DD

**示例**：
```
git-analyzer commits --since 2024-01-01 --until 2024-01-31
```

#### 7. analyze - 分析贡献度

**功能**：分析仓库的贡献度情况

**用法**：
```
git-analyzer analyze [path] [--since <date>] [--until <date>]
```

**参数**：
- `path`：仓库路径，默认为当前目录
- `--since`：开始日期，格式为YYYY-MM-DD
- `--until`：结束日期，格式为YYYY-MM-DD

**示例**：
```
git-analyzer analyze --since 2024-01-01
```

#### 8. report - 生成分析报告

**功能**：生成Markdown格式的仓库分析报告

**用法**：
```
git-analyzer report [path] [--output <file>] [--since <date>] [--until <date>]
```

**参数**：
- `path`：仓库路径，默认为当前目录
- `--output`：输出文件路径，默认为git_analysis_report.md
- `--since`：开始日期，格式为YYYY-MM-DD
- `--until`：结束日期，格式为YYYY-MM-DD

**示例**：
```
git-analyzer report --output analysis.md --since 2024-01-01
```

## 贡献度计算方法

贡献度计算基于以下指标：

1. **提交次数**：每个提交记2分
2. **文件变更数**：每个变更的文件记1.5分
3. **代码变更行数**：每行变更记0.5分
4. **提交价值**：基于以下因素计算：
   - 提交消息质量（长度、关键字）
   - 变更大小
   - 文件类型（代码文件权重高于文档文件）

总得分 = 提交次数得分 + 文件变更得分 + 代码变更得分 + 价值得分

## 报告内容

生成的Markdown报告包含以下内容：

1. **报告信息**：生成时间、分析仓库、时间范围
2. **贡献度排名**：按总得分排序的开发者贡献度列表
3. **最近提交记录**：最近10条提交的详细信息
4. **团队整体分析**：参与开发者数量、总提交次数、总文件变更数、总代码变更行数
5. **最活跃开发者**：贡献度最高的开发者信息

## 注意事项

1. 使用此工具前，确保已安装Git和GitPython库
2. 对于大型仓库，分析可能需要较长时间
3. 报告生成默认保存为Markdown格式，可直接在GitHub等平台查看
4. 支持的时间格式为YYYY-MM-DD

## 示例使用场景

1. **项目回顾**：分析团队在某个时间段的贡献情况
2. **绩效评估**：基于贡献度数据进行开发者绩效评估
3. **代码质量监控**：通过提交记录分析代码变更趋势
4. **团队协作分析**：了解团队成员的工作分布情况

## 故障排除

1. **GitPython未安装**：使用`pip install GitPython`安装
2. **仓库路径错误**：确保提供的路径是有效的Git仓库
3. **权限问题**：确保有足够的权限访问仓库
4. **网络问题**：克隆仓库时确保网络连接正常
